# UE 三视角渲染 View Profiles（v1）
生成日期：2026-01-08

目标：从同一份标准骨架主动画（UE Manny）稳定导出三种 2D 视角的预览/序列帧，并保证：
- 角色占屏比例稳定（不随动作漂移）
- pivot（锚点）稳定（建议脚底点）
- 背景可控（纯色或透明抠像友好）
- 输出可批处理（Movie Render Queue）

## 1. 通用场景与约束（所有视角共享）

### 1.1 场景
- 单独关卡：`L_ActionHub_Render`
- 地面平面：用于脚底对齐（可视化地平线）
- 光照：固定一套（尽量扁平、阴影可选；若要透明序列，可用纯色背景抠像或自定义渲染通道）
- 角色站位：世界原点附近，面向 +X（或你习惯的方向），必须固定

### 1.2 相机与锚点（Pivot）规则
推荐 pivot 规则：**脚底点**
- 在 Manny 上使用 `root/pelvis` 之外，再创建一个辅助 socket/marker（例如在 `foot_l` 与 `foot_r` 的地面接触点求中点作为脚底点）
- 最简做法：在渲染时固定角色的 root 在地面高度，并在后处理对齐 bbox_bottom_center

### 1.3 输出建议
- FPS：30
- 分辨率：先用 1024x1024（高清管线）或 512x512（中等），像素风可降到 256 高
- 序列命名：`<VIEW>/<category>/<action>/%04d.png`
- 预览视频：H.264 MP4，码率适中即可（仅用于检索）

## 2. VIEW_SIDE（横版侧视）

### 2.1 相机
- 类型：Orthographic（强烈推荐）
- 朝向：看向角色侧面（例如相机朝 -Y，取决于你的坐标系）
- Ortho Width：通过一次标定让“站立高度 ≈ 70% 画面高度”（后续固定）
- 位置/旋转：固定，不随动作变

### 2.2 方向策略
- 默认 `LR`：左/右通过镜像生成
- 仅在武器/盾牌明显不对称时，增加 `LR_no_mirror` 变体

### 2.3 投影约束建议（可选）
- 限制躯干绕纵轴扭转：`limit_torso_twist_deg = 25`
- 限制手臂绕轴扭转：`limit_arm_twist_deg = 45`

## 3. VIEW_TOP（俯视）

### 3.1 相机
- 类型：Orthographic
- 朝向：俯视向下（Pitch = -90° 或等效）
- Ortho Width：标定后固定
- 注意：俯视常需要更大的视野，避免手臂/武器出框

### 3.2 方向策略
- 推荐 `8dir`；最简可先 `4dir`
- 俯视动作“2D 友好”要点：
  - 减少躯干前后弯折过大导致投影怪
  - 避免大幅度的绕角色转身动作（可做视角变体）

## 4. VIEW_ISO45（45°等距/伪等距）

### 4.1 相机
推荐两种：
- **正交伪等距**（稳定、2D 友好）：Orthographic + 固定角度
- **透视伪等距**（更立体但更易漂）：Perspective + 固定焦距（不推荐做精灵图主线）

### 4.2 典型角度（可作为默认）
- Yaw：45°
- Pitch：35.264°（常用于等距感；你也可以按美术固定一个更“好看”的值）
- Roll：0
- 类型：Orthographic

### 4.3 方向策略
- 推荐 `8dir`
- 约束建议：
  - 限制躯干扭转：`limit_torso_twist_deg = 20`
  - 避免武器绕身穿插，必要时做 `@ISO45` 变体
